## Инструкция по начальной настройке программы
### Администратор должен назначить соответствующие роли пользователям системы.
1.	Специалист: Для сотрудников, обслуживающих клиентов. 
2.	Менеджер: Для сотрудников, распределяющих задачи специалистам;
3.	Бухгалтер ИТ фирмы: Для сотрудников бухгалтерии, работающим с первичной документацией;
4.	Кадровик расчетчик: Для сотрудников отдела кадров, а также сотруднику, занимающемуся расчетом заработной платы и планированием отпусков работников. 
### Данная система осуществляет  уведомление специалистов о назначенных работах в автоматическом режиме. 
1.	Создание бота: С помощью @BotFather необходимо создать своего бота и получить API-токен, сохранив его в соответствующую константу «Токен управления телеграм ботом».
2.	Требуется создать группу в Telegram и добавьте туда бота, как администратора группы.
3.	С помощью браузера или Postman выполните запрос https://api.telegram.org/bot[ВашТокен]/getUpdates. В полученном JSON найдите идентификатор группы, в которой получено сообщение, и сохраните его в соответствующую константу «Идентификатор группы для оповещения».
4.	Оповещения будут приходить в группу автоматически при создании нового документа «Обслуживание клиента», а также при изменении уже существующих документов.
### Заполнение оставшихся констант:
1.	Номенклатура работы специалиста – вид работ, для определения работ специалистов, выполняемых по заявкам от заказчиков.
2.	Номенклатура абонентская плата: вид услуги, означающий регулярную фиксированную оплату услуг заказчиком.
### Для совершения различных расчетов требуется обновить или добавить следующие данные:
1.	 В справочнике «Графики работы» должны быть добавлены кадровиком актуальные графики работ.
2.	Кадровиком вручную или с помощью обработки «Заполнение графиков» (Расчет управленческой зарплаты – Сервис – Заполнение графиков) заполняются рабочие и выходные дни.
3.	Для всех сотрудников в регистре «Условия оплаты сотрудников» (Расчет управленческой зарплаты – См. Также – Условия оплаты сотрудников)  должны быть установлены актуальные размеры окладов и процентов премий.
### Прочая информация:
1. Отчет «Анализ выставленных актов» показывает информацию, заполняемую через документы «Обслуживание клиентов» и «Реализация товаров и услуг».
2. У каждого контрагента, которому планируется оказание услуг специалистами, должны быть договоры с видом «Абонентское обслуживание».
3. Перед созданием документов по начислениям и выплатам зарплат и премий нужно заполнить документ/документы «График отпусков», чтобы учитывать возможный отпуск сотрудника фирмы.
4. Отчет «Выработка специалистов» обрабатывает данные заполняемые через документы «Обслуживание клиентов» (описание выполненных работ), «Начисление зарплаты».
5. Для формирования отчета «Расчеты с сотрудниками»  необходимо наличие документов по начислению фиксированных премий и зарплат, а также по выплатам зарплат.
6. Отчет «Расход запланированных отпусков» показывает информацию о запланированном отпуске и уже использованном его периоде.
### Подготовка БД для тестов:
1.	Загрузите в пустую базу файл выгруженной базы - Диплом.dt (она содержит необходимые начальные данные и соответствует разработанной конфигурации).
2.	В отдельной базе откройте Vanessa Automation.
3.	В качестве клиента тестирования укажите базу, в которую загружали Диплом.dt
4.	В параметрах запуска подключаемой базы укажите /N"Менеджер". Запустите сценарий  «ManagerTest». Это проверка планирования нескольких обслуживаний на специалистов от имени Менеджера.
5.	В параметрах запуска подключаемой базы укажите /N"Специалист". Запустите сценарий  «SpecTest». Это проверка закрытия обслуживаний от имени Специалиста.
6.	В параметрах запуска подключаемой базы укажите /N"Бухгалтер". Запустите сценарий  «BuhTest». Это проверка массового создания актов от имени Бухгалтера, а также проверка формирования отчёта «Анализ выставленных актов».
